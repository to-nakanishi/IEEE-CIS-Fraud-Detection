# IEEE-CIS Fraud Detection: 大規模決済データにおける不正検知モデルの構築と運用最適化
🔗 [Kaggle Competition Link](https://www.kaggle.com/competitions/ieee-fraud-detection)

## 1. プロジェクトの背景と目的
スーパーのレジで正当なカード決済が拒否（誤検知）されるのは顧客にとって気まずい瞬間であり、顧客体験を大きく損ないます。一方で、カード会社は背後で稼働する「不正利用防止システム」によって年間数百万ドルの被害を防いでいます。
本プロジェクトの目的は、大規模なeコマース取引データに対して機械学習モデルを適用し、**「不正検知の精度（被害防止）」と「誤検知の削減（スムーズな決済体験）」を高い次元で両立させること**です。

* **データ提供元**: Vesta Corporation（年間180億ドル以上の取引を保証する電子決済ソリューションの先駆者）
* **主催**: IEEE Computational Intelligence Society (IEEE-CIS)

## 2. アプローチの特徴
単なる予測精度の向上（Kaggleでのスコア追求）にとどまらず、**「実務での監視コスト」や「被害金額の最小化」といったビジネス要件**を想定したアプローチを採用しました。

1. **金額ベースのドメイン知識注入**
   取引を金額帯別に4つのセグメント（Bin0〜3）に分割し、それぞれの不正手口の特性に応じたモデル構築を検証。
2. **堅牢なバリデーション設計**
   時間経過に伴う不正パターンの変化を捉えるため、実運用に近い「時系列交差検証（TSCV：2カ月スライドウィンドウ）」を実装。

## 3. 主要な成果 (Results)
本プロジェクトでは、2つのアプローチでモデルを構築・比較検証しました。

| モデル | Public Score | Private Score | 実務上の特徴 |
| :--- | :--- | :--- | :--- |
| **全データ統合モデル（単一）** | **0.921** | **0.884** | 全体傾向の把握とモデルの保守・管理に優れる |
| **金額セグメント別モデル（4分割）**| **0.910** | **0.875** | 監視コストの最適化と高額被害の防遏（ぼうあつ）に優れる |

## 4. モデル比較と実務運用上の考察
当初、実際のビジネスにおける「損失金額の最小化」を目的とし、取引を金額帯別に4つのセグメント（Bin0〜3）に分割したアンサンブルモデルを構築しました。その後、ベースラインとして「全データ統合の単一モデル」を作成し、両者の比較検証を行いました。

### 【スコアにおける比較】
Kaggleのスコア上は、単一モデルが僅かに上回る結果となりました。
これは、セグメント別モデルにおいて「金額帯での分割」と「2カ月スライドウィンドウ（TSCV）」を掛け合わせたことで各セグメントの学習サンプルが減少し、局所的な過学習を起こしたためと考えられます。対して、単一モデルは全金額帯に共通する不正のサインを大局的に学習できていました。

### 【実務運用におけるセグメント別モデルのメリット】
スコアでは単一モデルに軍配が上がりましたが（Privateスコアの差は約0.9％）、実際の金融・決済実務にモデルを導入することを想定した場合、**金額セグメント別のアプローチには特筆すべき強力なビジネスメリットがある**と考えます。

1. **不正トレンドの解像度向上**
   金額帯ごとの不正手段の動向やトレンド変化を、ブラックボックス化させずに個別にモニタリングできる。
2. **監視リソースと費用対効果（ROI）の最適化**
   セグメント別に、Recall（再現率）とPrecision（適合率）の優先度を柔軟に調整できる。
   * *具体例: 件数自体は少ないが1件の被害額が甚大な「高金額帯」ではRecallを優先して監視の網を広げる。一方、件数が膨大な「低金額帯」ではPrecisionを優先し、目視確認などの人的な監視コストを抑えることで、組織全体の費用対効果を最大化できる。*

### 【結論】
モデルの構造的シンプルさや、運用・保守のコストを最優先する場合は「単一モデル」が適しています。一方で、限られた監視リソースを最適に配分し、ビジネス上の被害額をコントロールする観点では「セグメント別モデル」に大きな実務的優位性があります。
本プロジェクトを通じ、単純な予測精度だけでなく、**「実際の監視体制やビジネス上のKPI（被害抑制か、運用コスト削減か）に応じて、柔軟にモデルのアーキテクチャを選択・提案すべきである」**という重要な知見を得ました。
## 4. モデル選定のプロセスとビジネス的考察
当初、**ビジネス上の損失金額を最小化する目的**で、金額帯別に4つのセグメント（Bin0〜3）に分割したアンサンブルモデルを構築しました。しかし、ベースラインとして作成した「全データ統合の単一モデル」と比較した結果、スコアおよび実運用面において単一モデルが優位であると判断しました。

### 【結論と考察】
セグメント分割モデルは、金額帯での分割と2カ月スライドウィンドウを掛け合わせたことで各セグメントのサンプルサイズが減少し、局所的な過学習を起こした可能性が高いと考えられます。対して、単一モデルは全金額帯に共通する不正のサインを大局的に学習できていました。

一方で、**金額セグメント別のアプローチには、実務運用上以下の強力なビジネスメリットがある**ことを確認しました。
* Privateスコアにおける単一モデルとの精度差がわずか0.9％程度に留まること。
* 金額帯ごとの不正手段の動向・トレンド変化を個別に探れること。
* **セグメント別にRecall（再現率）とPrecision（適合率）の優先度を調整できること。**
  * *(例: 件数が少なく被害額が大きい「高額帯」はRecallを優先して監視の網を広げ、件数が膨大な「低額帯」はPrecisionを優先し監視コストを抑えることで、全体の費用対効果を最大化できる)*

最終的には、運用・保守の複雑性（分割ラインの策定やモデル管理コスト）を考慮し、**よりシンプルで堅牢な「単一モデル」を最終採択**としました。

## 5. 総評・今後の課題
本プロジェクトを通して、局所的にセグメント化した低金額層では一定の精度が得られたものの、全体を通して「分割点の最適な線の引き方」や「特徴量の追加不足」が浮き彫りとなりました。

特に、重要変数である `card1` 等については、どのセグメントでも一貫して寄与度が高かったため、単体としての利用だけでなく、他変数との組み合わせ（交互作用）を含めた特徴量生成をより深く検討すべきでした。初期のEDA段階で、金額や時間軸に関する特徴量を早期に追加し、軽量なモデルで試走させていれば、これらの重要性に早く着眼できたと考えます。

大規模データ（ビッグデータ）のハンドリングにおいて、処理の効率化や時間配分に課題を残しましたが、今後のプロジェクトでは「上流工程での仮説検証」と「前処理の最適化」をより強化し、ビジネス価値に直結する分析を追求していきます。
